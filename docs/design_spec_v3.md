# Fashion Bot — Design Spec v3 (21 марта 2026)

## Общий принцип
Один движок рендеринга (Satori), один формат (карточка-изображение), условная логика
по количеству фото и сегменту юзера. Placeholder-блоки в коллаже = антипаттерн.

## Два сегмента — два визуальных мира

| | Мама ребёнка (mom_girl/mom_boy) | Женщина для себя (no_kids) |
|---|---|---|
| Фон карточки | Тёплый розоватый `#F5EDE8 → #F0E8E4` | Холодный голубоватый `#E8EDF5 → #E4E8F0` |
| Цвет текста | `#5B3A4A` | `#3A4A5B` |
| Цвет muted | `#B09888` | `#8898A8` |
| Акцент | `#D080A0` | `#7090C0` |
| Тон Касси | Тёплый, про удобство: "классная пара!" | Стильный, про сочетание: "изысканное сочетание" |
| Кнопки | Надели / Переодень / Переслать | Надену / Другой / Stories (при 8+) |
| Прогресс порог | 8 вещей | 12 вещей |

## Три состояния карточки

### Состояние 1: "Погода" (0 фото в образе)
Когда в гардеробе 0 фото или недостаточно для образа.

**Layout (сверху вниз):**
1. Header: дата + контекст (садик/офис) + имя
2. Погода: большая иконка (☀️/🌧/⛅) + температура утро→день→вечер
   - Вечерняя температура красным `#C05050` если дождь/похолодание
3. Блок рекомендаций:
   - **Мама:** иконки-сетка слоёв: 🩲👚👖🧥👟🧢 + подписи ("базовый", "кофта", "штаны"...)
     Акцентный фон `#E8F0FF` + border для обязательных (куртка, зонт)
   - **Женщина:** формула образа в ряд: "Водолазка → Блейзер → Тренч → Брюки"
     Без жаргона (не "base/mid/outer"), конкретные вещи с иконками
4. Комментарий Касси: совет привязан к погоде
5. CTA: "📸 Сфоткай вещи → соберу образ!"
6. Кнопки: [📸 Сфоткать] [Потом]

**Размер карточки:** 440×620px (стандартный Satori)

### Состояние 2: "Гибрид" (1-7 фото в образе)
Когда есть реальные фото, но не все слоты заполнены.

**Layout:**
1. Header: дата + контекст + имя + погода компактно (справа)
2. Реальные фото вещей — крупные карточки:
   - 2 фото: два рядом, height ~90px
   - 3-4 фото: 2 сверху + 1-2 снизу поменьше
   - 5-7 фото: масштабируется, крупные → мелкие по важности
   - Каждая карточка: фото (base64) + пастельный градиент по цвету + подпись снизу
   - borderRadius: 10px
3. Блок "Для полного образа:" (если есть недостающие слоты)
   - Мелкие иконки в ряд: 🧢🧣 с подписями. Фон пастельный по типу вещи.
   - Текст мягкий: "Для полного образа:" (НЕ "ДОБАВЬ" — звучит как вина)
4. Строка "🩲 колготки · майка" мелким текстом по центру
5. Комментарий Касси
6. Прогресс-бар:
   - Мама: "4/8 · 📸 Сфоткай шапку!" (milestone: "Ещё 2 → полный образ!")
   - Женщина: "5/12 · ещё 7 → полноценный стилист!"
   - Скрыть при 0 фото (пустая полоска демотивирует)
   - Цвет бара: акцент сегмента
7. Кнопки: [👍 Надели] [🔄 Переодень] [📸 Шапку]
   - Третья кнопка = конкретная CTA "📸 {вещь}" вместо "Переслать" (нечего шарить)

### Состояние 3: "Полный" (8+ фото, все слоты заполнены)
Полноценный flat lay из реальных фото.

**Layout:**
1. Header: дата + контекст + имя + погода компактно
2. Flat lay: фото вещей в свободном размещении (position absolute)
   - Иерархия размеров: верхняя одежда крупнее → верх/низ средние → обувь/акс мелкие
   - Примерная раскладка:
     - Куртка: top center, ~130×95px
     - Кофта: left, top+70px, ~110×90px
     - Штаны: right, top+76px, ~108×90px
     - Обувь: left bottom, ~72×56px
     - Шапка/шарф: right bottom, ~60×48px
   - Каждая карточка: фото + подпись (overlay снизу, полупрозрачный чёрный)
3. Палитра-кружочки: правый верхний угол, 3 цвета по 10×10px
4. Строка "🩲 колготки · майка" мелким
5. Комментарий Касси (стилистический, объясняет ПОЧЕМУ):
   - Мама: "Зелёная кофта — давно не надевали! С синими джинсами свежо 🌿"
   - Женщина: "Лаванда + бежевый тренч — нежная весенняя палитра ✨"
6. Кнопки: [👍 Надели/Надену] [🔄 Переодень/Другой] [📤 Переслать/Stories]
   - "📤 Stories" только для no_kids
   - "📤 Переслать" только для мам

**Нет прогресс-бара.** Нет CTA фото. Нет "для полного образа".

## Обработка фото: async RMBG

Текущая проблема: RMBG ~4 сек, silueta ~1.5 сек. 4 сек = мама думает "зависло".

**Решение: async pipeline.**
1. Мама отправляет фото
2. Мгновенно (0с): "✨ Анализирую..." + typing indicator
3. Vision (~2с): "👚 Вижу кофту розовую! Сохраняю..."
4. Ответ юзеру: "✅ Кофта розовая добавлена! (3/8)"
5. RMBG в фоне (worker queue, HIGH priority):
   - Получает item_id + photo_file_id
   - Скачивает фото → RMBG → сохраняет processed_photo_url
   - К вечернему брифу фон уже удалён
6. Если RMBG упал → коллаж с оригинальным фото (fallback)

Юзер НЕ ждёт bg removal. Feedback мгновенный.

## Satori: что починить в server.mjs

### 1. Шрифты
Загрузить ВСЕ из fonts/:
```javascript
const FONTS = [
  { name: 'Nunito', data: readFileSync('./fonts/Nunito-Regular.ttf'), weight: 400, style: 'normal' },
  { name: 'Nunito', data: readFileSync('./fonts/Nunito-Bold.ttf'), weight: 700, style: 'normal' },
  { name: 'DejaVu', data: readFileSync('./fonts/DejaVuSans.ttf'), weight: 400, style: 'normal' },
];
```
Проверить что Nunito покрывает кириллицу. Если нет → DejaVu основной, скачать DejaVuSans-Bold.

### 2. Emoji через Twemoji
```javascript
loadAdditionalAsset: async (code, segment) => {
  if (code === 'emoji') {
    // Check local cache first
    const cached = emojiCache.get(segment);
    if (cached) return cached;
    // Fetch from Twemoji CDN or local file
    const cp = [...segment].map(c => c.codePointAt(0).toString(16)).join('-');
    // Try local file first, then CDN
    const svgData = await loadEmojiSVG(cp);
    emojiCache.set(segment, svgData);
    return svgData;
  }
}
```
Скачать 20 нужных SVG в renderer/emoji/ как fallback если сеть недоступна.
Emoji: ☀️⛅🌧🌨🌤💨✨👧👩🧥👚👖👟🧢🧣🧤🧦🩲🌂💍👢👜

### 3. Цветные точки = CSS div, не emoji
Палитра-кружочки: `{ width: 10, height: 10, borderRadius: '50%', background: HEX }`.
Цвет из item.color через COLOR_HEX маппинг (уже есть в style_config.py).
