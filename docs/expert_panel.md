# Fashion Bot — Совет экспертов (динамический)

## Принцип
Состав совета меняется в зависимости от контекста задачи. Не все эксперты
нужны для каждого решения. Добавлять/убирать по релевантности.

## Пул экспертов

### UX / Дизайн
- 👗 **Стилист** (15 лет, personal styling, Stitch Fix опыт) — комбинации, цвет, ценность стилистического совета. Когда: дизайн коллажей, комментарии Касси, стайл-квиз.
- 🧠 **UX лид** (12 лет, mobile, бывш. Headspace/Calm) — сканируемость, клики, Telegram-ограничения. Когда: любой UI/UX вопрос.
- 📸 **Fashion-блогер** (120K подписчиков) — shareability, тренды визуала, viral potential. Когда: шеринг, stories, визуальный контент.

### Пользователи (ЦА)
- 👩‍👧 **Маша** (28, мама Алисы 3г, визуал) — утренняя спешка, реальные потребности мамы. Когда: любая фича для мам.
- 👩 **Лена** (26, no_kids, офис, "полный шкаф — нечего надеть") — стиль, $9 vs $200, re-roll, occasions. Когда: любая фича для женщин.
- 👨 **Дима** (35, папа, "жена скинула ссылку") — вторичная ЦА, family sharing. Когда: v2.0 family features.

### Психология
- 👶 **Детский психолог** (18 лет) — утренние ритуалы, одевание без конфликта, вовлечение ребёнка. Когда: UX для мам, коллаж для детей.
- 🧩 **Поведенческий психолог** — привычки, gamification, nudges, variable reward, commitment. Когда: retention, onboarding, milestones, trial.

### Бизнес / Продукт
- 🎯 **Product Manager** (B2C subscription, 10 лет) — приоритизация, scope creep, что вырезать. Когда: роадмап, sprint planning.
- 📊 **Growth PM** (mobile apps) — воронка, метрики, activation, retention, A/B. Когда: метрики, конверсия, onboarding flow.
- 💰 **Monetization specialist** (subscriptions, paywalls) — paywall timing, pricing, free vs premium. Когда: тиры, trial, pricing.

### Техника
- ⚙️ **Solo-founder CTO** (1 person, bootstrapped) — реальные оценки effort, техдолг, что 2 часа vs 2 недели. Когда: роадмап, архитектурные решения.

## Как выбирать состав

| Контекст задачи | Обязательные | Опциональные |
|---|---|---|
| UX/дизайн экрана | UX лид, Маша, Лена | Стилист, блогер, дет.психолог |
| Роадмап/приоритизация | PM, Growth, CTO | Маша, Лена, monetization |
| Монетизация/pricing | Monetization, Growth, Лена | PM, Маша |
| Онбординг | UX, повед.психолог, Growth, Маша, Лена | PM |
| Коллаж/бриф | Стилист, UX, Маша, Лена | Блогер, дет.психолог |
| Retention/engagement | Повед.психолог, Growth, Маша, Лена | PM |
| Маркетинг/viral | Блогер, Growth, Лена | Monetization |
| Техническое решение | CTO | UX (если влияет на UX) |

## Формат ревью
Каждый эксперт даёт:
1. Одобрение / критику конкретных решений
2. Что упущено (blind spots)
3. Конкретную рекомендацию (не абстрактную)

Не нужно опрашивать всех по каждому вопросу — это шум. 3-5 самых
релевантных экспертов на задачу.
